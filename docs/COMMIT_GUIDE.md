# Git 提交规范

## 核心规则

- **语言**：使用简体中文，使用祈使句（如“添加”而非“添加了”），中英文间用空格隔开
- **长度**：标题≤50字符
- **标点**：末尾不用句号

## 格式

```
标题

正文

尾部
```

- **标题（必填）**：`<类型>(<范围>): <描述>`（范围可选）
- **正文（可选）**：空一行后写变更背景、原因，每行使用无序列表格式（`-`）
- **尾部（可选）**：关联Issue（如 `Closes #123`）或破坏性变更（`BREAKING CHANGE:`）

### 类型定义

- `feat`：新功能
- `fix`：Bug修复
- `docs`：文档变更
- `style`：代码格式调整（不影响功能）
- `refactor`：重构（非功能/非修复）
- `perf`：性能优化
- `test`：测试相关
- `chore`：构建工具/辅助脚本变更

## 优秀示例

```
docs(CODE_WIKI): 更新架构图为 Mermaid 格式，移除重复内容
```

```
refactor(api): 简化 HTTP 请求拦截器逻辑

- 移除冗余的错误处理代码，统一使用 Axios 的标准拦截机制
```

```
feat(user): 添加 OAuth2 第三方登录支持

- 集成 GitHub 和 Google 的认证库
- 在登录页增加“使用第三方账号登录”按钮
- 处理回调并创建新用户会话

Closes #1024
```

## 禁止事项

- 模糊描述（如“更新代码”“修了个bug”）
- 错误语气（如“添加了”“修复了”）
- 多余前缀/签名（直接输出提交内容）
